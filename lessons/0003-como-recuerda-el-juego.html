Cómo recuerda el juego

_process(delta) se repite, pero cada frame no empieza desde cero. Las variables conservan lo ocurrido y forman el estado actual de la partida.

Victoria tangible: rastrear una variable real desde su valor inicial hasta su representación en pantalla.

1. Código no es lo mismo que estado

Código: instrucciones que permanecen escritas, como “sumá delta”.

Estado: valores que cambian mientras jugás, como “llevamos 37.4 segundos”.

Una variable es un nombre asociado a un valor. En este proyecto, tiempo_partida funciona como una memoria: cada frame lee el tiempo acumulado, suma delta y guarda el nuevo resultado.

La interfaz dibuja el reloj.
var tiempo_partida := 0.0      # nace

func _process(delta):
    tiempo_partida += delta    # cambia y recuerda

etiqueta_tiempo.text = ...     # se representa

En GDScript, var declara una variable. El operador += significa “tomá el valor actual, sumale lo de la derecha y guardá el resultado”. La referencia oficial de GDScript documenta variables y operadores.

5. Recuperá el flujo

Idea que debe quedar

El estado es la memoria actual del juego. Una regla lee esa memoria, la transforma y guarda un estado nuevo; la interfaz representa una parte de ese resultado.
